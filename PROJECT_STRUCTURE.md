# Project structure — HexaStack Solutions main site

This repo is the **HexaStack Solutions company website** (hexastacksolutions.com): services, blog, contact, work, SEO location pages. It is **not** a product-specific site (e.g. not the HexaBill marketing site only). Agent docs in `docs/agent-system/` are **dynamic skills** reusable for the main site and any product or campaign.

---

## Commands

- **Full stack (dev):** `npm run dev` — Vite + Express (concurrently)
- **Frontend only:** `npm run client`
- **Backend only:** `npm run server`
- **Build:** `npm run build` — Prisma generate + Vite build → `dist/`
- **Preview:** `npm run preview`
- **DB:** `npm run db:push`, `npm run db:generate`, `npm run db:seed`, etc.

---

## Removed / not used

- **api/** — Redundant duplicate backend (was for Vercel); removed. API is only `server/` + Netlify Functions.
- **db/** — Legacy folder (contained `custom.db`). App uses `server/db.ts` (Prisma + MongoDB) only; root `db/` removed.
- **dist/** — Build output; regenerated by `npm run build`. In `.gitignore`; delete anytime to clean.
- **vercel.json** — Removed; deployment is Netlify-only for this repo.
- **Google domain push** — We do not push to Google domain. SEO is on-site (titles, meta, sitemap). GSC/GBP are optional; see `docs/SEO_AND_CLEANUP.md`.

## Cleanup plan (applied)

1. Single SEO scope doc: `docs/SEO_AND_CLEANUP.md` — no dependency on GSC submission.
2. No extra repo folders: no `api/`, no root `db/` or `files/`, no project-level .mjs (only in node_modules). `dist/` gitignored.
3. Agent/skills: `.cursor/AGENTS.md` and `.cursor/skills/README.md` are the index.
4. Root config files kept: `package.json`, `package-lock.json`, `tsconfig.*.json`, `vite.config.ts`, `eslint.config.mjs`, `postcss.config.mjs`, `components.json`, `netlify.toml`, `index.html` — all required for build and run. `.md` (README, PROJECT_STRUCTURE) and `docs/` kept.
