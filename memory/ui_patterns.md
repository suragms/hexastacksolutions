# UI patterns

Successful UI patterns. Update after UI/UX or Development stages.

---

- **Minimal layout:** Clear hierarchy; one primary CTA per section; avoid clutter.
- **Responsive:** Mobile-first; breakpoints at 768px and 1024px; single column on mobile, 2–3 columns on desktop where useful.
- **Sections:** Hero → Problem → Solution → Features → Use cases → Social proof → CTA → Footer. Reuse for landing pages and product templates.
- **Forms:** Labels above fields; one primary button; loading state on submit.
